第一周：基础热身
Day 1：认识 Shell 与第一个脚本
学习要点

什么是 Shell、终端、命令行

常见 Shell：bash、zsh

创建脚本文件：touch myscript.sh

Shebang：#!/bin/bash

赋予执行权限：chmod +x myscript.sh

运行脚本：./myscript.sh 或 bash myscript.sh

实操任务

打开终端，创建一个名为 hello.sh 的文件。
在文件第一行写入 #!/bin/bash，第二行写入 echo "Hello, World!"。
保存文件，并给它添加执行权限。
运行脚本，观察输出。
修改脚本，让它打印当前日期（提示：date 命令）。
Day 2：变量与命令替换
学习要点

定义变量：name="value"（等号两边无空格）

使用变量：$name 或 ${name}

环境变量：$HOME、$PATH

命令替换：$(command) 或 `command`

实操任务

编写脚本 info.sh，定义变量 username 为你的系统用户名（或通过 whoami 获取）。
打印 "当前用户是：$username"。
用命令替换获取当前工作目录，存入变量 current_dir，并打印。
再获取当前 Shell 的类型（echo $SHELL），一并输出。
尝试将脚本中的 $username 改为 ${username}，验证结果相同。
Day 3：位置参数与特殊变量
学习要点

$0：脚本名称

$1、$2……：第1、第2个参数

$#：参数个数

$@：所有参数（列表）

$?：上一条命令的退出状态（0 表示成功）

实操任务

编写脚本 greet.sh，接收一个名字作为参数（$1）。
如果参数个数不为 1，则提示“用法：./greet.sh 你的名字”，并退出。
否则打印 "你好，$1！"。
测试：分别运行 ./greet.sh（无参数）和 ./greet.sh Alice。
在脚本中加上一行 echo "脚本名称是：$0"，观察输出。
Day 4：用户交互（read）
学习要点

read 命令：从标准输入读取数据

常用选项：-p（提示信息）、-t（超时）、-s（静默输入）

实操任务

编写脚本 ask.sh，使用 read -p 询问用户“你最喜欢的编程语言是什么？”。
将答案存入变量 lang，然后打印 "原来你喜欢 $lang 啊！"。

用 read -t 5 添加超时：如果5秒内没有输入名字，就自动退出。

Day 23：开机启动与 systemd
学习要点

常用命令：systemctl enable、systemctl start、systemctl status

实操任务

创建一个脚本 /usr/local/bin/startup_demo.sh，内容为 echo "Started at $(date)" >> /tmp/startup.log。
使用传统方法：在 /etc/rc.local 的 exit 0 前添加一行 /usr/local/bin/startup_demo.sh（需要 root 权限）。重启，检查 /tmp/startup.log。
使用 systemd 方法：创建 /etc/systemd/system/startup-demo.service，内容：
text
[Unit]
Description=Startup Demo

[Service]
ExecStart=/usr/local/bin/startup_demo.sh
Type=simple

[Install]
WantedBy=multi-user.target
执行 sudo systemctl enable startup-demo 和 sudo systemctl start startup-demo，检查状态和日志。
